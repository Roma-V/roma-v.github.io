What I stand for

While learning web development I started analyzing web content and decided to create a short list of common requirements that I see important and often times find lacking on web sites.

Mobile friendliness

As of now I mostly use mobile devices for searching internet and looking through new content. At the same time I see throngs of people with smartphones around and realize that most of them make do with desktop computers only occasionally. Thus mobile friendly responsive design is key in modern world.

In addition to content layout a usual problem is touch support on web pages. Sometimes an interface is hard to use on mobile - that is bearable, though frustrating. Sometimes a website is developed with desktop users in mind and there is no way to use some features with touchscreen - because conventional mouse events are used which might have no duplicating touch counterparts.

Accessibility

I like going through learning materials but do not have a lot of time for it. Thus it often happens on my way in metro, where I prefer listening to material instead of watching or reading. My best friend is a screen reader: even though far from perfect it helps me a lot. The problem arising is accessibility of pages. The screen reader helps me seeing how little some page owners care about their content being accessible.

Where I am and where I go

I am learning. I have some programming background and therefore working through language features is not a problem. My current focus is on general principles of web design and getting practice. Here is the list of resources I have been using and recommend them to others:

  • MDN front-end web developer (in process) - one of the best resources covering the very basics. Stretching beyond the course and investigating the contents of numerous links provided throughout the course one finds endless resources for learning. Therefore the course gives both instant knowledge and references for future.
  • Full stack open - a great course covering little bits of everything. It includes both frond-end and backend projects. Even though it does not give a lot of depth, there are a lot of links to learn more on each topic. While the material is great, it is based on a number of frameworks and requires some prior knowledge of HTML, CSS and JS basics. Otherwise one can learn how to use a library but end up seeing it as a black box. My recommendation is to go for this course after learning some "vanilla" ways of doing staff.

What I do

Here go my projects. There is not a lot to show. Hence I include some planned items.

  • Stopwatch - I like keeping track of how much time I spend on things and doing scrum-like sprints for long running tasks. I was frustrated with some native timers as well as online ones. Finally decided to make my own version that is available online and use the minimum but essential functionality.
  • Calculator - the same story as the stopwatch. Once upon a time I had to work on restricted Windows system with no calculator and no reasonable way to install it. There are decent online options. However poor keyboard support sometimes might drive me mad. Therefore decided I need to make one that suits my needs.
  • List of blog posts - this is a small web app created during FullstackOpen course. It consists of React-based front-end, Node.js back-end and cloud based MongoDB database. Also included are some basic tests and Github Actions for testing and deployment. I use it occasionally to save some links to interesting posts for reading some time in the future.
  • To do list - I mostly use Trello, but some simple reminder list would not hurt.
  • Shooting game (WIP) - a game prototype made with Phaser engine. The idea came when I looked through various game jams and got really impressed with "One Button Jam" rules: making a game controlled with a single button. At first it seemed too restrictive, however seems like I managed to squeeze myself into. The game is playable, but requires some further refinements in order to count as a real game.
  • Draggable text - a whiteboard with stickers. Inspired by English CLub TV show where you are presented with a collection of words shown on the screen as a random sequence and asked to construct a meaningful sentence. I made a similar board where one can input text which is then split into words, the words are shuffled and presented. The app was meant to help my wife with her online classes. She tried some native apps, but they were either not a good fit for the task or did not support Russian language.
